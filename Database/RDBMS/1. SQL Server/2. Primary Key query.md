## Find All Primary Keys

```sql
SELECT
    *
FROM
    sys.key_constraints
WHERE
    type = 'PK';
```

---

## Search Specific Primary Key(s)

```sql
SELECT
    *
FROM
    sys.key_constraints
WHERE
    type = 'PK'
    AND name LIKE '%Product%';
```

---

## Primary Key on Which Table

```sql
SELECT
    kc.name AS PrimaryKeyName,
    t.name AS TableName
FROM
    sys.key_constraints kc
INNER JOIN
    sys.tables t
        ON kc.parent_object_id = t.object_id
WHERE
    kc.type = 'PK';
```

---

## Specific PK on Which Table

```sql
SELECT
    kc.name AS PrimaryKeyName,
    t.name AS TableName
FROM
    sys.key_constraints kc
INNER JOIN
    sys.tables t
        ON kc.parent_object_id = t.object_id
WHERE
    kc.name = 'PK_Products';
```

## PK Uses Which Column(s)

```sql
SELECT
    kc.name AS PrimaryKeyName,
    t.name AS TableName,
    c.name AS ColumnName
FROM
    sys.key_constraints kc
INNER JOIN
    sys.index_columns ic
        ON kc.parent_object_id = ic.object_id
        AND kc.unique_index_id = ic.index_id
INNER JOIN
    sys.columns c
        ON ic.object_id = c.object_id
        AND ic.column_id = c.column_id
INNER JOIN
    sys.tables t
        ON kc.parent_object_id = t.object_id
WHERE
    kc.type = 'PK';
```

## PK of a Specific Table

```sql
SELECT
    kc.name AS PrimaryKeyName,
    c.name AS ColumnName
FROM
    sys.key_constraints kc
INNER JOIN
    sys.index_columns ic
        ON kc.parent_object_id = ic.object_id
        AND kc.unique_index_id = ic.index_id
INNER JOIN
    sys.columns c
        ON ic.object_id = c.object_id
        AND ic.column_id = c.column_id
WHERE
    kc.type = 'PK'
    AND OBJECT_NAME(kc.parent_object_id) = 'Products';
```

## Something Important Every .NET Developer Should Know

When EF Core creates:

```csharp
public Guid Id { get; set; }
```

and migration generates:

```csharp
table.PrimaryKey("PK_Products", x => x.Id);
```

SQL Server actually creates **two things**:

### 1. Primary Key Constraint

```sql
PK_Products
```

stored in:

```sql
sys.key_constraints
```

### 2. Backing Unique Index

Usually:

```sql
PK_Products
```

stored in:

```sql
sys.indexes
```

That's why if you run:

```sql
SELECT *
FROM sys.indexes
WHERE name = 'PK_Products';
```

you'll often find the PK there too.

## My SQL Server Metadata Investigation Flow

```text
Primary Keys      -> sys.key_constraints
Foreign Keys      -> sys.foreign_keys
Indexes           -> sys.indexes
Columns           -> sys.columns
Tables            -> sys.tables
FK Columns        -> sys.foreign_key_columns
Index Columns     -> sys.index_columns
```

For EF Core migration debugging, these **7 system views** are the ones you'll use repeatedly when reviewing generated `CreateTable`, `CreateIndex`, `AddForeignKey`, `DropIndex`, and `DropForeignKey` operations.

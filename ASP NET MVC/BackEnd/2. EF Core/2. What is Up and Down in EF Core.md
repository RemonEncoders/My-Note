## 🟢 EF Core migration file-এর structure মনে রাখো

প্রতিটা migration file-এ সাধারণত দুইটা method থাকে:

```csharp
public partial class SomeMigration : Migration
{
    protected override void Up(MigrationBuilder migrationBuilder)
    {
        // Apply changes (going forward)
    }

    protected override void Down(MigrationBuilder migrationBuilder)
    {
        // Revert changes (rollback)
    }
}
```

---

## 🟡 এখন তোমার প্রশ্ন: `RenameColumn` কোথায় থাকবে?

👉 **Answer: BOTH Up এবং Down এর মধ্যে**

---

## 🟢 1. Up method (forward migration)

এখানে থাকবে নতুন change apply করার logic:

```csharp
protected override void Up(MigrationBuilder migrationBuilder)
{
    migrationBuilder.RenameColumn(
        name: "PhoneNumber",
        table: "Users",
        newName: "Mobile");
}
```

---

## 🔴 2. Down method (rollback)

এখানে থাকবে আগের অবস্থায় ফেরানোর logic:

```csharp
protected override void Down(MigrationBuilder migrationBuilder)
{
    migrationBuilder.RenameColumn(
        name: "Mobile",
        table: "Users",
        newName: "PhoneNumber");
}
```

---

## 🧠 খুব গুরুত্বপূর্ণ ধারণা

👉 Up = "apply change"  
👉 Down = "undo change"

তাই rename মানে:

- Up এ forward rename
    
- Down এ reverse rename
    

---

## ⚡ Real-world intuition

ধরো:

```
PhoneNumber → Mobile
```

তাহলে:

### Up:

👉 PhoneNumber থেকে Mobile

### Down:

👉 Mobile থেকে PhoneNumber

---

## 🟡 EF Core কেন দুই জায়গায় লাগে?

কারণ migration system হলো **reversible system**

মানে তুমি চাইলে:

```bash
dotnet ef database update PreviousMigration
```

তাহলে EF Core Down() চালাবে।

---

## 🧠 এক লাইনে সারাংশ

👉 RenameColumn method **Up এবং Down দুই জায়গাতেই থাকে**  
👉 Up = forward rename  
👉 Down = reverse rename

## 🟢 1. আসলে যখন তুমি column rename / add / delete করো কী হয়?

যখন তুমি ASP.NET Core এ model change করো, তারপর চালাও:

```bash
dotnet ef migrations add SomeChange
```

EF Core তখন তোমার **আগের model snapshot** আর **নতুন model** compare করে।

তারপর decide করে কী SQL operation দরকার।

---

## 🔴 2. EF Core কীভাবে migration generate করে?

### ✅ (A) Add Column করলে

Model এ নতুন property যোগ করলে:

```csharp
public string Phone { get; set; }
```

Migration তৈরি হয়:

```csharp
migrationBuilder.AddColumn<string>(
    name: "Phone",
    table: "Users",
    nullable: true);
```

---

### ❌ (B) Remove Column করলে

Property delete করলে:

```csharp
DropColumn("Phone", "Users");
```

---

### ⚠️ (C) Rename করলে (এটাই tricky part)

যদি তুমি এটা করো:

```csharp
// আগে
public string PhoneNumber { get; set; }

// পরে
public string Mobile { get; set; }
```

EF Core **না বুঝে এটা rename**, বরং ভাবে:

👉 পুরানো column delete  
👉 নতুন column add

তাই generate করে:

```csharp
DropColumn("PhoneNumber", "Users");
AddColumn<string>("Mobile", "Users");
```

👉 এতে data loss হয়!

---

## 🟡 3. EF Core কি আলাদা Rename method আছে?

👉 **হ্যাঁ, আছে!**

EF Core migration builder এ method আছে:

```csharp
migrationBuilder.RenameColumn(
    name: "OldName",
    table: "Users",
    newName: "NewName");
```

---

## 🔴 কিন্তু সমস্যা কী?

👉 EF Core **auto-detect করে rename বুঝতে পারে না**

তাই:

- তুমি column rename করলেও
    
- EF Core সাধারণত `Drop + Add` বানায়
    

---

## 🟢 4. তাহলে RenameColumn কখন use হয়?

তুমি manually migration edit করলে:

```csharp
migrationBuilder.RenameColumn(...)
```

বা তুমি explicit ভাবে command ব্যবহার করলে (advanced tooling/adjustment এর মাধ্যমে)

---

## 🧠 5. Best practice (real world)

যখন rename করো, তখন তোমাকে করতে হয়:

```bash
dotnet ef migrations add RenamePhoneToMobile
```

তারপর migration file খুলে নিজে change করো:

❌ EF-generated:

```csharp
DropColumn();
AddColumn();
```

✅ Fix it to:

```csharp
RenameColumn();
```

---

## ⚡ এক লাইনে সারাংশ

👉 EF Core default ভাবে rename বুঝতে পারে না, তাই drop/add করে  
👉 কিন্তু `RenameColumn()` method আছে  
👉 তুমি চাইলে migration file manually edit করে safe rename করতে পারো

---

চাও হলে আমি তোমাকে real production scenario দেখাতে পারি যেখানে rename ভুল করলে data loss হয় এবং কীভাবে safe migration করা হয়।

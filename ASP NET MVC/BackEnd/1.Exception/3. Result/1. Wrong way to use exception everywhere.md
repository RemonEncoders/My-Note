# ১. পুরানো Exception Flow

আগে কী হতো?

---

## Service

```csharp
public User GetUser(int id)
{
    var user = _db.Users.Find(id);

    if(user == null)
        throw new Exception("User Not Found");

    return user;
}
```

---

## Controller

```csharp
try
{
    var user = _service.GetUser(id);

    return Json(user);
}
catch(Exception ex)
{
    return Json(new
    {
        success = false,
        message = ex.Message
    });
}
```

---

# Flow

```text
	Failure
	 ↓
	THROW
	 ↓
	CATCH
	 ↓
	Convert to JSON
	 ↓
	Frontend
```

# ৬. Why Result Pattern Better Here?

কারণ এখন controller clearly বুঝতে পারছে:

```csharp

	if(!result.IsSuccess)
	
```

মানে:

# failure is EXPECTED

No emergency।

# ৭. Massive Mental Shift

Exception style:

```text

```

```text

```

# ৯. Advanced Result Pattern

Production systems এ শুধু message না।

More structured data থাকে।

---

Example:

```csharp
public class Result<T>
{
    public bool IsSuccess { get; set; }

    public string ErrorCode { get; set; }

    public string Message { get; set; }

    public T Value { get; set; }
}
```

---

# Controller

```csharp
return Json(new
{
    success = false,
    code = result.ErrorCode,
    message = result.Message
});
```

---

# Frontend

Frontend now can do:

```javascript
if(response.code === "USER_NOT_FOUND")
{
    showToast(response.message);
}
```

Much cleaner architecture।

# ১০. Exception vs Result in UI Perspective

---

# Exception

```text
System Failure
```

Usually:

```json

```

---

# Result

```text
Business Failure
```

Usually:

```json

404 Not Found
409 Conflict
```

# ১১. Modern API Design

```text

```

---

# Unexpected Failure

Throw exception।

Middleware handles:

```text

```

# ১৩. Example Complete Flow

---

# Service

```csharp
public async Task<Result<User>> GetUser(int id)
{
    var user = await _db.Users.FindAsync(id);

    if(user == null)
        return Result.Fail<User>("User not found");

    return Result.Ok(user);
}
```

---

# Controller

```csharp
var result = await _service.GetUser(id);

if(!result.IsSuccess)
{
    return NotFound(new
    {
        success = false,
        message = result.Error
    });
}

return Ok(result.Value);
```

---

# Frontend Receives

```json
{
   "success": false,
   "message": "User not found"
}
```

# ১৪. তাহলে Exception কোথায় use হবে?


এটা business failure না।

এটা system failure।

তখন:

```text

```

Middleware catches।

Returns:

```json

   "message": "Internal Server Error"
}
```


# Real-world systems কী করে?

## Solution 1 — Short-lived Access Token

Example:

```text
JWT expires in 15 minutes
```

Risk কমে।

---

## Solution 2 — Refresh Token

- Access token short-lived
    
- Refresh token DB/Redis এ থাকে
    

Logout করলে refresh token revoke করা হয়।

---

## Solution 3 — Token Blacklist

Redis/database এ revoked token রাখা হয়।

কিন্তু:

- আবার `stateful` হয়ে যায় partially

# Practicalভাবে কী শেখাচ্ছে

ওরা probably practicalভাবে দেখাচ্ছে:

```text
Multiple containers
+ Load Balancer
+ JWT auth
```

তারপর:

- request different server এ যাচ্ছে
    
- তবুও auth working
    

যাতে developer বুঝে:

```text
Modern distributed systems
cannot depend on in-memory session
```

# কিন্তু enterprise systems এ pure JWT alone enough না

Real systems usually use:

```text
JWT
+ Redis
+ Refresh Tokens
+ API Gateway
+ Identity Provider
```

কারণ:

- logout
    
- revoke
    
- device management
    
- token rotation
    
- security monitoring
    

এসব দরকার হয়।

---

# সবচেয়ে important architectural lesson

ওরা আসলে শেখাচ্ছে:

```text
State is enemy of scalability
```

যত বেশি:

- server memory dependency
    
- local session
    
- local cache
    

থাকবে,

তত distributed scaling hard হবে।

তাই modern cloud-native architecture চায়:

```text
Stateless Services
Shared Storage
Distributed Cache
Token-based Identity
```

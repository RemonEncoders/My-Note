
# আগে traditional monolith কী করতো

আগে সাধারণত:

```text
Server Memory Session
```

ব্যবহার হতো।

ধরো:

```text
User Login
→ Server creates Session
→ Session stored in RAM
```

তারপর browser cookie পাঠায়:

```text
ASP.NET_SessionId=XYZ
```

এখন request এলে server RAM থেকে session খুঁজে পায়।

---

# Problem কোথায়?

ধরো এখন:

```text
Nginx
   ↓
Server 1
Server 2
Server 3
```

User প্রথম request দিল:

```text
→ Server 1
```

Session save হলো:

```text
Server 1 RAM
```

কিন্তু next request গেল:

```text
→ Server 3
```

এখন Server 3 session খুঁজে পাবে না।

কারণ:

- session only Server 1 RAM-এ ছিল
    

ফলে user suddenly logout হয়ে যাবে।

এটাই:

# “Logout Paradox”

---

# Traditional workaround — Sticky Session

Nginx configure করা হয়:

```text
Same user → Same server
```

এটাকে বলে:

```text
Sticky Session
```

---

# কিন্তু এটা bad solution কেন?

কারণ:

- load imbalance হয়
    
- server crash করলে session lost
    
- scaling difficult
    
- cloud-native না
    

তাই modern distributed system এটা avoid করে।
